Em busca de saber o comportamento dos alunos de Programao 1 da Universidade Federal de Campina Grande, foram feitas analises nas submisses das atividades que os alunos submetem ao longo do perodo. Como a populao dos alunos de programao 1 muito grande, utilizei uma amostra para fazer inferncias em relao aos alunos de programao 1 e as atividades. Para este estudo, coletamos as submisses dos alunos de Programao I da Universidade Federal de Campina Grande (UFCG) durante o perodo letivo 2014.2. No total, h 15148 submisses de 101 alunos para 427 questes.

Ao longo da disciplina, cada aluno dita o seu ritimo. O aluno resolve questes e a medida que vai acertando vai subindo de nvel e recebendo mais questes. A correo das atividade feita de forma automtica. Aps a submisso de uma atividade o aluno informado se a resposta enviada a correta ou no, caso no seja a resposta esperada, o aluno pode reenviar quantas vezes achar necessrio.

Por essa razo, existem questes que possuem um grande nmero de submisses. Ento achei importante descobrir a mdia de submisses das 5 questes com maior nmero de submisses.

As questes que possuim mais subsmisses so:

Devido ao intervalo de confiana no podemos afimar qual questo que possuim a maior mdia no nmero de submisses. Podemos apenas afimar que a questo vida_collatz a que possui a menor mdia dentre as 5 primeiras.

Foi obserevado tambm que existe uma grande varincia no nmero de submisses por aluno. Para efeito de comparao dividimos os alunos em 2 grupos: Um grupo com poucas submisses (Menos de 150 submisses durante todo o perodo) um grupo com muitas submisses (Mais de 150 submisses durante todo o perodo)

O valor 150 no foi escolhido arbritariamente, ele representa o valor mdio de submisses da nossa amostra.

Como os alunos vo passando de nveis ao longo da disciplina, achei importante saber (em mdia) quantas submisses so necessrias at que o aluno acerte a questo.

Foi ento calculado a mdia do nmero de acertos da amostra para os dois grupos de alunos. Com 95% de precisso temos o seguinte resultado.

alunos <- file %>% 
  select(student) %>%
  table() %>%
  as.data.frame()

colnames(alunos) <- c("Aluno", "Submissoes")
alunosOrder <- alunos[order(-alunos$Submissoes),] 

condicao1 = head(alunosOrder, n=39)$Aluno
condicao2 = tail(alunosOrder, n=62)$Aluno

muitasSubmissoes <- file %>%
  filter(student %in% condicao1)

poucasSubmissoes <- file %>%
  filter(student %in% condicao2)

muitasSubmissoes["Submissoes"] <- "Muitas Submisses"
poucasSubmissoes["Submissoes"] <- "Poucas Submissoes"

fileSubmissoes <- union(muitasSubmissoes, poucasSubmissoes) %>%
  filter(result == "True")

ggplot(fileSubmissoes, aes(x = Submissoes, y = attempt)) + 
  stat_summary(fun.y = mean, geom = "point") + 
  stat_summary(fun.data = mean_cl_normal, geom = "errorbar", colour = "blue", width = 0.2) +
  labs(y='Mdia de submisses') + 
  theme(axis.text.x = element_text(angle = 45, hjust = 1), panel.background=element_blank())

possvel notar que os alunos que possuem muitas submisses, demoram mais submisses para responder a questo corretamente. O que significa dizer que, os alunos que tem muitas submisses no necessariamente respondeu muitas questes corretamente.
